# UX Flow & Screen Design

## 1. 정보 구조도 (Information Architecture)

* 앱 메인 (App Root)
  * 어린이 모드 (기본 화면)
    * 홈 화면 (내 레벨, 그랜드 퀘스트, 미션 카드 더미)
    * 내 지갑 (구매한 보상 카드 보관함)
    * 포인트 상점 (부모가 진열한 상품 구매)
    * 타이머 화면 (집중 모드, 플레이 모드)
  * 부모 인증 게이트 (PIN 4자리 입력)
    * 부모 관리 모드
      * 그랜드 퀘스트 대시보드 (추가, 승인)
      * 상점 대시보드 (상품 추가, 가격 수정, 활성화 토글)

## 2. 어린이 모드 핵심 화면 플로우

### 2.1 홈 화면 (Home)
* 상단 (장기 비전): 아이의 프로필, 현재 레벨(EXP 바), 진행 중인 그랜드 퀘스트 달성률 표시.
* 중앙 (퀵 액션): 내 지갑 보기(현재 보유 카드 수) / 상점 가기(보유 포인트) 버튼 배치.
* 하단 (미션 시작): 자주 하는 미션 카드 더미. 카드를 슬와이프 후 화면 중앙으로 드래그 시 즉시 집중 타이머 가동.

### 2.2 집중/플레이 타이머 (Timer UI)
* 집중 모드: 딥 블루 톤. 중앙에 거대한 25분 타이머. 일시정지와 포기하기 버튼만 존재. 완료 시 폭죽 이펙트와 함께 포인트 획득 모달 노출.
* 플레이 모드: 시간형 카드 사용 시 진입. 네온 오렌지 톤의 신나는 UI. 종료 1분 전 화면 테두리 점멸로 종료 예고. 종료 시 "다음 미션을 준비할 시간이에요!" 전체 화면 팝업 노출.

### 2.3 포인트 상점 및 지갑 (Shop & Wallet)
* 상점: 보유 포인트 내에서 활성화된 상품의 구매 버튼 활성화. 터치 시 지갑으로 카드가 날아가는 애니메이션 제공.
* 지갑 사용 경험:
  * 타이머형 카드(게임): 터치 즉시 플레이 타이머 시작.
  * 실물형 카드(과자): 터치 시 부모님께 기기를 보여주세요 팝업 노출. 부모가 수령 확인 버튼을 누르면 카드가 파쇄되는 이펙트 발생하며 삭제.

## 3. 부모 관리 모드 핵심 화면 플로우

### 3.1 진입 및 보안
* 진입: 홈 우상단 설정 아이콘 터치 -> 하단에서 숫자 키패드 팝업. 
* 인증: 4자리 PIN 입력 후 검증 통과 시 진입 (연속 오류 시 타임아웃 페널티 적용).

### 3.2 상점 관리 (Shop Admin)
* 디지털 환율 셋업: "게임 10분 = 10 포인트" (스태퍼 기호로 직관적 조절 기능 제공).
* 실물 보상 덱 빌딩: 실물 보상 추가 버튼 터치 -> 아이콘 선택 -> 이름 입력(예: 포카칩) -> 가격 설정 및 저장.
* 리스트 뷰: 생성된 모든 카드가 리스트업 되며, 우측 토글 스위치로 실시간 상점 진열 여부(ON/OFF) 제어.

### 3.3 그랜드 퀘스트 관리 (Grand Quest Admin)
* 추가 플로우: 목표 텍스트 입력 -> 달성 조건 설정(포모도로 누적 N회 또는 레벨 N 달성) -> 실물 보상 텍스트 입력(예: 놀이공원 가기).
* 승인 플로우: 아이가 조건을 달성하면 리스트에 승인 대기 뱃지 점멸. 항목을 터치하여 최종 보상 지급을 완료 상태로 전환.

## 4. 핵심 UX 가이드라인
1. 숫자보다 직관적인 메타포: 잔여 시간을 숫자로 계산하게 하지 말고, 카드 1장 = 1블록의 물리적 감각으로 치환.
2. 명확한 시작과 끝: 공부 타이머와 게임 타이머를 동일한 UX 문법으로 구성하여, 노는 것도 스스로 통제하고 종료하는 훈련 유도.
3. 가시적 성취감: 카드가 지갑에 꽂히는 모션, 사용 시 파쇄되는 모션, 레벨업 시 전체 화면 전환 이펙트 등을 통해 보상 회로(도파민 루프)를 건전하게 자극.
